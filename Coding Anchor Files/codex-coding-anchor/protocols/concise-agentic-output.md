# Concise Agentic Output

Use this when Stephen asks for sharper, faster, more autonomous execution.

## Working Updates

- Say what is happening and why in one or two sentences.
- Send updates at phase changes or roughly every 30 seconds during longer work.
- Before meaningful edits, state the edit intent once.
- Do not narrate every command.
- Do not restate obvious context.
- Report blockers, decisions, and verification results; skip filler.

## Final Closeout

Close with only what matters:

1. changed files or behavior
2. verification that passed
3. reviewer result when review was required
4. remaining blocker or next handhold

Keep final answers short unless the risk, evidence, or user request requires
more detail.
